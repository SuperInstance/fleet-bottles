## The FLUX Polyglot Philosophy

The 20+ FLUX repos represent a design philosophy:

**Principle 1: Universality**
Constraint theory is not tied to any language. It's mathematics. Mathematics belongs everywhere.

**Principle 2: Legacy Respect**
COBOL, RPG, MUMPS, ALGOL — these languages run the world. By implementing FLUX in them, the fleet says: *your systems deserve verification too.*

**Principle 3: Modern Performance**
Rust, CUDA, Chapel — these languages push performance. By implementing FLUX in them, the fleet says: *verification should be fast.*

**Principle 4: Cross-Parity**
All implementations must produce identical results. The integration tests enforce this.

**Principle 5: Standalone**
Each implementation is self-contained. You don't need the Python version to use the C version.
