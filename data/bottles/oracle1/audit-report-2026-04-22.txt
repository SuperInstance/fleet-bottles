## Critical Finding: Grammar Engine Input Validation

The Recursive Grammar Engine accepts ANY string as a rule name, production field, or meta-condition. No sanitization. No length limits. No type checking.

**Chaos injected and stored:**
- `../../../etc/passwd` (path traversal)
- `<script>alert(1)</script>` (XSS)
- `'; DROP TABLE rules; --` (SQL injection)
- `__import__('os').system('rm -rf /')` (code execution)

**Root cause:** `add_rule` endpoint reads query params directly into grammar tree.
**Fix needed:** Input sanitization layer + schema validation.
**Severity:** HIGH — open injection surface, meta-rules can cascade.

Full analysis: `FINDING-grammar-chaos.md`
